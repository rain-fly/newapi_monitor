## Why

当前需要持续监控 NewAPI 提供的模型 API 可用性和响应延迟，确保在每天工作时间（8:00-18:00）内能够及时发现服务中断或性能下降问题。传统的人工检查方式无法做到实时监控和历史趋势分析，需要一个自动化系统来完成这项工作。

## What Changes

- 新增 NewAPI 模型可用性监控服务，按设定间隔自动测试 API 可用性和延迟
- 新增数据记录模块，将每次测试结果（时间戳、可用状态、延迟、错误信息）持久化存储
- 新增可视化看板页面，展示多时间窗口（3分钟/30分钟/3小时/6小时/12小时/24小时）的可用性统计
- 新增定时调度功能，仅在每天 8:00-18:00 执行监控任务

## Capabilities

### New Capabilities

- `api-health-monitor`: 核心监控能力，包括：
  - 定时调度器（每3分钟触发一次测试）
  - 时间窗口过滤（仅在8:00-18:00执行）
  - 最小token测试（用最少的输入/输出token测试模型可用性和延迟）
  - 结果数据持久化（存储测试记录）
- `availability-dashboard`: 数据展示能力，包括：
  - 多时间窗口可用性计算（3分钟/30分钟/3小时/6小时/12小时/24小时）
  - 实时状态展示
  - 历史趋势可视化
- `newapi-adapter`: NewAPI 接口适配能力，包括：
  - NewAPI API 调用封装
  - 错误处理和重试机制
  - 响应解析和延迟计算

### Modified Capabilities

- （无）

## Impact

- 新增依赖：需要接入 NewAPI 服务
- 数据存储：使用本地 SQLite/JSON 文件存储测试记录
- 看板展示：可作为独立 Web 页面或集成到现有监控系统
